Introduction

The graduation work is part of "LAB-REM FACET-UNT", a program that hopes to incorporate FACET-UNT as a node in the Collaborative Network of Remote Laboratories of CONFEDI. The objective of this program is to design, build and implement the remoteness of experimental practices in different areas of knowledge in engineering careers at FACET-UNT.

To carry out this program, we work in two teams. Hardware team and software team.

  • Hardware team, formed by us, Electronic Engineering students, in charge of the design, development and implementation of the remotization of the different experimental practices.
  • Software team formed by students of the Computer Engineering career, is responsible for the design and implementation of the web application and user authentication.
Labs

We can present the following laboratories.

Digital systems:

  • UART communication test
  • I2C communication test.

Telecommunications:

  • Point-to-point WiFi link.
  • Point-to-point link with software defined radio.

Digital signal processing and control:

  • Subsampling, aliasing and strobe effect.
  • Automatic position control system

Basic experimental physics:

  • Converging lens testing.
  • Diverging lens testing.
Problems to resolve

Currently, FaCET has several on-site laboratories for students, teachers and researchers who are members of our faculty. During the period of pandemic and isolation, many students were unable to perform the laboratories due to the non-existence of remote laboratories and the impossibility of attending in person to the faculty facilities.

The implementation of this project allows a student from any of the universities that make up the Collaborative Network to access virtually, through a web page, to perform different laboratory practices that are physically installed in our faculty, also provides access to the manipulation of different tools and obtaining results once the experience is completed.

Protocol

The communication procedure is as follows. Every time you want to modify some data in the laboratory. The Server must send a...

POST /HTTP/1.1 + (JSON a enviar)

Json to send (example)

{"Estado": [ 0, true, false], "Llaves": [ false, false ], "Analogico": [ 100, 200 ]} {"Estado": [ 0, true, false], "Llaves": [ false, false ], "Analogico": [ 100, 200 ]} {"Estado": [ 0, true, false], "Llaves": [ false, false ], "Analogico": [ 100, 200 ]}

And to obtain data previews or data reporting you must send a:

Json to receive (example)

{"Estado": [ 0, true, false], "Llaves": [ false, false ], "Analogico": [ 100, 200], "Error": 0}
Error Type Value
Error-free 0
Frequency error 1
Value error 2

Digital Systems

Syntax

Estado: It is an array consisting of 3 elements in the following order: [Laboratory, SubLab, Start of experiment].

Selected Laboratory Lab num SubLab Start of the experiment Experiment status
UART communication test 0 false false Finalized
I2C communication test 0 true false Finalized
UART communication test 0 false true Initialized
I2C communication test 0 true true Initialized

Nota:Los valores booleanos se los puede recibir en cualquier forma, es decir, true=1=HIGH y false=0=LOW

Speed: It is an integer type data that indicates the desired communication speed. *e.g.: 9600, 115200, etc.

Elements per Laboratory - UART Laboratory

  • Utiliza Cyclone II
  • Pulsadores: [Pulsador1, Pulsador2,Pulsador3,Pulsador4 ]
  • Indicadores: [indicador1, indicador2, indicador3, indicador4]
    • Los mismos son utilizados para representar en la API lo que se ve por la WebCam.
